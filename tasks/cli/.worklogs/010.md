## 2026-03-07T20:18:00Z

Implemented the HTTP handler with request routing and directory listing.

**Approach:** Split handler into two files — `handler.go` for routing/dispatch and `directory.go` for listing logic. Used existing packages (classify, markdown, render) for file type detection and rendering.

**Changes:**
- `internal/handler/handler.go` — `Handler` struct with `ServeHTTP` that routes requests based on path type (markdown, text, directory, binary)
- `internal/handler/directory.go` — Directory listing with sorted entries (dirs first, then files, both alphabetically case-insensitive)
- `internal/handler/handler_test.go` — 14 tests covering all routing paths, sort order, parent links, URL encoding, 404s
- `internal/render/types.go` — Added `HasParent` bool to `directoryData` to fix parent link display for root-child directories
- `internal/render/templates/directory.html` — Updated template to use `HasParent` instead of `ParentHref` for conditional

**Key decision:** Fixed a bug in the render package where `{{if .ParentHref}}` failed for subdirectories whose parent is root (empty string is falsy in Go templates). Added a `HasParent` bool field to properly control parent link visibility.
