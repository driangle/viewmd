## 2026-03-08T10:30:00Z

Implementation complete.

**Changes made:**

1. **`.github/workflows/release.yml`** — Rewrote the release workflow:
   - Injects version via `-ldflags "-X main.version=${VERSION#v}"` so `viewmd --version` reports the release tag
   - Creates `.tar.gz` archives per platform (matching taskmd pattern)
   - Generates `.sha256` checksum files per archive
   - Uploads both tarballs and checksums as release assets
   - Added `update-homebrew` job that downloads checksums, generates the formula, and pushes it to `driangle/homebrew-tap` via GitHub API

2. **`apps/cli/cmd/viewmd/main.go`** — Changed `const version` to `var version` so ldflags can override it at build time

3. **`README.md`** — Added Homebrew installation section, fixed `go install` path

4. **Formula template** — Follows exact same pattern as `taskmd.rb` in the tap (platform-specific URLs, SHA256 checksums, binary rename in install)

**Requires:** A `HOMEBREW_TAP_TOKEN` secret in the viewmd repo with write access to `driangle/homebrew-tap`

**Remaining:** Manual testing of `brew install driangle/tap/viewmd` after first release with the new workflow

**Open items:**
- The "Test brew install" subtask cannot be completed until a release is created with the new workflow
- Windows binaries are still built but not included in the Homebrew formula (Homebrew doesn't support Windows)
