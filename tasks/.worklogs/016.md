## 2026-03-08T11:00:00Z

Completed VitePress documentation site setup.

**Approach:** Followed the same pattern as the taskmd docs site (driangle.github.io/taskmd/) — same VitePress config structure, custom theme with brand colors, and GitHub Pages deployment workflow.

**Key decisions:**
- Used green brand color (#4a9e6e) to differentiate from taskmd's blue
- Placed docs at `docs/` (not `apps/docs/`) since viewmd is simpler
- Used `npm` instead of `pnpm` (no monorepo workspace)
- Docs deployment triggers on push to main when docs/ changes

**Created files:**
- `docs/package.json` — VitePress project with docs:dev/docs:build scripts
- `docs/.vitepress/config.ts` — Nav, sidebar, social links, search, footer
- `docs/.vitepress/theme/` — Custom theme with green brand colors
- `docs/index.md` — Homepage with hero + 6 feature cards
- `docs/getting-started/` — Introduction, Installation, Quick Start
- `docs/guide/` — Viewing Files, Directory Browsing, Search, Keyboard Shortcuts
- `docs/reference/` — CLI Flags, Configuration, Supported File Types
- `.github/workflows/docs.yml` — GitHub Pages deployment workflow
- `.gitignore` — Added VitePress build output exclusions

**Verified:** `npm run docs:build` produces a static site without errors.
